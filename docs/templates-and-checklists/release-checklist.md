# Release Checklist

Run this checklist before publishing updates to dashboards, datasets, or documentation.

- [ ] Requirements captured using the [Dashboard Documentation Template](dashboard-template.md).
- [ ] SQL queries reviewed for correctness and performance.
- [ ] Data validation performed against trusted sources (Grafana logs or control totals).
- [ ] Permissions verified for target audience.
- [ ] Screenshots or GIFs captured if UI changes are significant.
- [ ] Documentation updated with change log and version/date.
- [ ] Links to Discord Gateway API events noted for new data fields.
- [ ] Accessibility review completed (color contrast, labels).
- [ ] Post-release monitoring plan defined (dashboards, alerts).
